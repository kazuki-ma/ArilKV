## 1. Fast map: where command definitions come from

1. Command metadata source: `redis/src/commands/*.json`
- Example: `redis/src/commands/get.json` (`"function": "getCommand"`)
- Example: `redis/src/commands/replicaof.json` (`"function": "replicaofCommand"`)

2. Generated command table: `redis/src/commands.def`
- Header says auto-generated by `redis/utils/generate-command-code.py`.
- Contains `redisCommandTable[]` entries with command name, flags, arity, and function pointer.

3. Runtime registration: `redis/src/server.c`
- `populateCommandTable()` loads `redisCommandTable[]` into runtime dictionaries (`server.commands`, `server.orig_commands`).
- `lookupCommand*()` resolves argv to `struct redisCommand`.

## 2. Command execution pipeline (important context)

Inbound to execution path:
- Socket read: `redis/src/networking.c` `readQueryFromClient()`
- RESP parse and pending command queue: `processInputBuffer()`
- Dispatch wrapper: `processCommandAndResetClient()`
- Main command gate/checks: `redis/src/server.c` `processCommand()`
- Actual function execution and propagation/stats: `redis/src/server.c` `call()`

What `processCommand()` checks before running `cmd->proc`:
- command existence / arity
- auth and ACL
- cluster routing / cross-slot checks
- OOM checks
- replica read-only gate (`replica-read-only` behavior)
- stale/loading/busy conditions
- transaction queueing (`MULTI` context)

What `call()` does around `cmd->proc(c)`:
- latency / slowlog / command stats updates
- monitor feed
- propagation decision (AOF / replica stream) via `alsoPropagate(...)`

## 3. Runtime context objects to know before implementing commands

Primary structs:
- `redis/src/server.h` `struct client`
- `redis/src/server.h` `struct redisCommand`

`client` fields most relevant to command handlers:
- request args: `argc`, `argv`
- selected db: `db`
- protocol version: `resp`
- resolved command pointers: `cmd`, `lastcmd`, `realcmd`
- execution / role flags: `flags` (`CLIENT_*`)
- replication-related fields: `replstate`, `reploff`, `repl_ack_off`, `replid`, etc.

`redisCommand` fields most relevant:
- `proc` (actual function pointer)
- `arity`
- `flags` (`CMD_WRITE`, `CMD_READONLY`, `CMD_ADMIN`, `CMD_STALE`, ...)
- key spec metadata (`key_specs`, `getkeys_proc`)

Command flag definitions are in `redis/src/server.h` (CMD_* macros).
